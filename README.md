# BOJ Step Coach

백준 온라인 저지(BOJ) 단계별 문제풀이를 **올해 안에 클리어**하는 것을 목표로, 체계적인 학습 코칭을 제공하는 Claude Code 스킬입니다.

문제 번호만 입력하면 풀이 환경이 자동으로 세팅되고, 연간 계획 수립부터 진도 추적, 단계별 힌트, 코드 리뷰, 반례 분석까지 문제풀이의 전 과정을 지원합니다.

---

## 주요 기능

### 1. 풀이 파일 자동 생성

문제 번호를 입력하면 즉시 풀이를 시작할 수 있는 환경을 만들어 줍니다.

- [acmicpc.net](https://www.acmicpc.net)에서 문제 정보(제목, 제한, 예제 입출력)를 자동으로 가져옵니다.
- 풀이 템플릿(`solution.py` 등)과 테스트 코드(`test_solution.py`)를 함께 생성합니다.
- 예제 입출력은 물론 문제 특성에 맞는 엣지 케이스도 테스트에 포함됩니다.
- 기존 프로젝트의 디렉토리 구조, 언어, 입출력 패턴, 테스트 러너를 자동 감지하여 동일한 패턴으로 생성합니다.
- 지원 언어: **Python, TypeScript, C++, Java, JavaScript, Kotlin**

### 2. 연간 학습 계획 생성

"계획 세워줘"라고 말하면 현실적인 주간 계획을 수립합니다.

- 오늘부터 12월 31일까지 남은 기간을 기반으로 주간 단위 계획을 생성합니다.
- 단계별 난이도를 반영해 초반은 빠르게, 후반은 여유 있게 배분합니다.
- 버퍼 주간을 포함해 번아웃 없이 완주할 수 있는 목표를 제시합니다.
- `plan.json`으로 저장되어 진도 추적과 항상 연동됩니다.

### 3. 진도 추적 및 피드백

"잘 하고 있어?", "진도 알려줘"로 현재 상태를 확인합니다.

```
📊 진도 현황

🗓️ 현재: 2026년 4월 9일 (15주차)
📈 전체 진행률: ██████░░░░ 62% (186/300문제)

이번 주 목표
- 단계: step15_백트래킹
- 목표 문제: 5문제 중 3문제 완료 ✅

컨디션 체크
- ✅ 계획 대비 1일 앞서가고 있습니다!
- 💡 평균 풀이 시간: 45분/문제
- 📊 최근 정답률: 80%
```

- 문제 시작, 테스트 통과, 제출 확인 시점에 `progress.json`이 자동 업데이트됩니다.
- 연속 풀이 기록(streak), 평균 풀이 시간, 정답률 등 성과 지표를 추적합니다.

### 4. 단계적 힌트 시스템

"모르겠어", "힌트 줘"라고 말하면 **정답을 바로 주지 않고** 단계별로 도움을 줍니다.

| 레벨 | 내용 | 예시 |
|------|------|------|
| Lv.1 | 방향 제시 | 핵심 인사이트를 질문으로 유도 |
| Lv.2 | 알고리즘 공개 | 사용할 알고리즘 이름과 개념 설명 |
| Lv.3 | 접근법 설명 | 의사코드 수준의 풀이 전략 |
| Lv.4 | 핵심 코드 조각 | 가장 어려운 부분만 3~5줄 |
| Lv.5 | 전체 풀이 | 3회 이상 요청 시에만, 한글 주석 포함 |

### 5. 코드 리뷰

"다 풀었어", "리뷰해줘"라고 말하면 테스트 실행 후 리뷰를 제공합니다.

- 시간/공간 복잡도 분석 및 제한 시간 내 통과 여부 판단
- 잘한 점에 대한 구체적 피드백
- 가독성, 효율성, 안전성, 관용적 표현 관점의 코드 개선 조언 (개선 전/후 비교 포함)
- 핵심 정리 및 유사 문제 추천

### 6. 반례 자동 생성

"틀렸어", "반례 찾아줘"라고 말하면 코드를 분석해 놓친 케이스를 찾아줍니다.

- **경계값**: 입력 범위의 최솟값/최댓값, 0, 음수, 매우 큰 입력
- **특수 케이스**: 빈 입력, 모든 원소 동일, 이미 정렬된 입력, 중복 값
- **함정 케이스**: 자료형 오버플로우, 출력 형식 오류, off-by-one 에러

실패하는 케이스만 보여주고 원인은 알려주지 않아 스스로 디버깅하도록 유도합니다. 발견한 반례는 테스트 파일에 자동 추가됩니다.

### 7. README 자동 생성

계획 수립, 진도 변경, 문제 완료 시 풀이 루트의 `README.md`를 자동으로 생성/업데이트합니다. 전체 진행률, 주간 계획표, 현재 페이스 등을 마크다운 테이블로 정리합니다.

### 8. 문제 완료 시 자동 업데이트

테스트가 모두 통과하면 코드 리뷰 전에 다음을 자동으로 처리합니다:

- `plan.json`의 해당 문제를 완료 처리하고 카운터를 업데이트
- 해당 단계의 모든 문제가 풀리면 상태를 `clear`로 변경
- `weekly_plan`에서 완료된 문제를 제거
- README 진도표 갱신

## 설치 방법

### 방법 1: 직접 복사

이 저장소의 `.claude/skills/boj-step-coach/SKILL.md` 파일을 자신의 프로젝트에 동일한 경로로 복사합니다.

```bash
# 프로젝트 루트에서
mkdir -p .claude/skills/boj-step-coach
cp /path/to/SKILL.md .claude/skills/boj-step-coach/SKILL.md
```

### 방법 2: 저장소 클론

```bash
git clone https://github.com/min-99/boj-step-coach.git
cd boj-step-coach
```

## 사용법

스킬이 설치된 상태에서 Claude Code와 자연어로 대화하면 됩니다.

```
# 새 문제 시작
"1000번 풀자"
"#15649 파일 만들어줘"

# 계획 및 진도
"올해 계획 세워줘"
"진도 알려줘"
"잘 하고 있어?"

# 도움 요청
"모르겠어, 힌트 줘"
"어떻게 접근하면 돼?"

# 풀이 완료 후
"다 풀었어, 리뷰해줘"
"맞는지 확인해줘"

# 오답 디버깅
"틀렸어, 반례 찾아줘"
"왜 안 되는지 모르겠어"
```

## 프로젝트 구조

스킬이 기본으로 제안하는 디렉토리 구조입니다. 기존 프로젝트에 풀이가 있으면 그 패턴을 자동으로 감지하여 따릅니다.

```
boj/
├── README.md                       # 전체 진도 요약 (자동 업데이트)
├── progress.json                   # 문제별 풀이 기록
├── plan.json                       # 연간 학습 계획
├── step01_입출력과_사칙연산/
│   ├── p1000_A+B/
│   │   ├── solution.py             # 풀이 코드 (사용자 작성)
│   │   └── test_solution.py        # 테스트 코드 (자동 생성)
│   └── ...
├── step02_조건문/
│   └── ...
└── ...
```

## 데이터 파일

### plan.json

연간 학습 계획을 저장합니다. 목표, 단계별 진행 상황, 주간 계획이 포함됩니다.

```json
{
  "goal": "2026년 12월 31일까지 백준 단계별 문제풀이 전체 클리어",
  "created_at": "2026-04-09",
  "total_steps": 40,
  "weekly_plan": [
    {
      "week": 1,
      "dates": "4/9 - 4/15",
      "step": "step01_입출력과_사칙연산",
      "problems": [1000, 1001, 1008],
      "estimated_hours": 2,
      "difficulty": "쉬움"
    }
  ]
}
```

### progress.json

문제별 풀이 기록과 전체 통계를 추적합니다.

```json
{
  "last_updated": "2026-04-09",
  "problems": {
    "1000": {
      "step": "step01",
      "status": "accepted",
      "started_at": "2026-04-09T10:00:00",
      "solved_at": "2026-04-09T10:15:00",
      "attempts": 1,
      "time_spent_minutes": 15,
      "language": "python",
      "hints_used": 0
    }
  },
  "stats": {
    "total_solved": 1,
    "total_problems": 300,
    "current_step": 1,
    "streak_days": 1,
    "avg_time_minutes": 15
  }
}
```

## 설계 철학

| 원칙 | 설명 |
|------|------|
| **정답을 주지 않는다** | 5단계 힌트 시스템을 통해 스스로 풀도록 유도합니다 |
| **자동화로 마찰을 줄인다** | 파일 생성, 테스트, 진도 기록을 자동화해 풀이에만 집중하게 합니다 |
| **현실적인 목표** | 난이도와 기간을 고려한 주간 계획으로 번아웃 없이 완주를 돕습니다 |
| **기존 프로젝트 존중** | 언어와 구조를 강요하지 않고 사용자의 기존 패턴을 감지해 따릅니다 |
